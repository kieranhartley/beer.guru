1.1 Describing the purpose of IT in modern business.

In order to create a successful business, you need information on which to base your decisions. This is just as true today as it was centuries ago, successful businesses would have gathered, analysed and acted upon various information. Imagine a brewery in 1500AD England. It would need to know how many sales they are expected to generate based on the previous year figures. They would need information from the hop farms and barley farms to forecast the supply of raw ingredients. They would need to work out their fixed & variable costs, yields, wastage and profit margins. These bits of information may have arrived in many forms. They could be general business surveys. Letters from suppliers or vendors. Internal records from previous years. Anecdotes passed on via word of mouth from customers and publicans. Published articles and books. The more information they made available to themselves, the more successful the business could become. In the modern age, this is just as true, but businesses now have many more sources of information, more ways to gather that information and an ever-increasing number of ways to process it. Hospitals have access to all different kinds of equipment which can perform scans of the human body and output it into electronic form. Police departments no longer need entire rooms to file and store data, they can fit this information onto a pocket-sized hard drive. It should be noted however, that with these increases in technology, come risks to data security. Advancements in Information Technology mean that we now have access to: quicker communication, electronic databases, more robust storage and retrieval novel ways to capture data and novel ways to process and analyse that data. If we focus just on digital IT businesses, some of the main advancements in technology that we have seen in the last decade or so include: increased processing power of CPUs & GPUs, faster and more robust storage, faster broadband, wireless technology, cloud technology, machine learning (AI) and cryptography.

Method

To create a digital project first you must ensure that each member of the team has access to the same software that is going to be used to create the project. If it involves coding, access to the same development environment tool is crucial to ensure that each member of the team can collaborate on the project effectively. On this project we used Visual Studio Code. It is one of the most popular source-code editors in the world due to being open-source and free. Some of the features that make VSC even more appealing are the built in support for git and Github, allowing you to seamlessly keep track of your different versions and upload them to be shared. In addition to this, VSC has many extensions & plug-ins that streamline functions between other software that you may be using on the project. These extensions help to improve your productivity. LiveServer allows you to host a local server which updates any changes to your code in real-time. Prettier enforces a single style of code formatting, which saves time spent formatting the code ‚Äì this is especially useful when multiple people are coding the same project. VSC even has a Slack Chat extension which makes it easier to share files, and allows you to embed a chat box into the environment which is useful for chatting to your team whilst sharing your screen and coding.

Communication tools

Working on digital projects opens the door to collaborating with team mates whilst working from entirely different locations. With this in mind, whilst working remotely online, it becomes much more important to use communication tools to ensure teams are still working closely together, avoiding errors, and not wasting time. On the project we have used Slack. Not only does it allow clear communication. It also allows communications to be directed to different public channels, private channels and direct messages. It also allows for the uploading of files, sharing code using code blocks and acts as a directory of people working on a team. Slack essentially functions as an archive of an entire project. If channels are used correctly it is possible to go back through and check the progress of the entire project.

Project Management

Similarly, to communication, it is very important to have a robust Project Management system when working remotely and collaborating with teams over the internet. This ensures that tasks can be split up and delegated to team members, allows all team members to view the progress of a project, lets completed tasks be archived and new tasks can be allocated. We used Trello, which allows you to break the project into many smaller tasks, quickly allocate them, and ensure that the entire team is working towards the same goal.

1.4 Describe the risks that might impact digital projects ‚Äì what causes digital projects to fail? Why?

Lack of Prioritisation

If priorities are not set in stone then the project may start to suffer. If the entire team are not aware of the key priorities of the project then time could be wasted on smaller, less important modules instead of focusing on the main elements ‚Äì it could even be totally innocent and due to lack of guidance.

Poor communication between Project Manager & teams

The project manager acts as the middle man/woman between the clients/stakeholders needs and wants and the issues raised by their team. If the project manager does not give out clear advice, and ask for regular feedback then the project may begin to go off track. The project manager needs to interpret the client‚Äôs requests, and guide the team on how to do it. However, it is very important to note that the team of coders are just as important as they are working on the front line and with their point of view, they will be able to spot problems or issues before they arise. It is important that teams are given chance to air their concerns without being ignored or criticised.

Poor communication between Project Manager & clients or stakeholders

Project managers also need to be able to speak freely to the clients or stakeholders and not have to fear criticism for being honest. One frequent problem seen in the tech industry and creative industries is that clients will expect far more work than their budget allows. The project manager needs to be able to manage their expectations and inform them of what their budget will allow, or advise them to increase their budget. Regular check-ins with clients should be conducted so that the client can see the progress of the project, be made aware of any issues or problems, and change the direction of the project if necessary.

1.7 Describe legal guidelines and constraints that impact digital projects ‚Äì How does GDPR impact digital projects?

Removing Net Neutrality

There has been lots of controversy around this topic, especially in the United States. Net Neutrality is the principle that all websites should be treated equally in regards to bandwidth and resources. The suggested idea is that this neutrality could be removed and large corporations could pay money to have access to more bandwidth and better search engine results ‚Äì effectively squashing smaller companies websites out of the market. If this idea was to take off it could change the internet as we know it. This would have serious implications to any digital project that was half way through production, as any plans related to bandwidth and search engine optimisation would have to be adjusted. It would also require changes to the business model of the project as it may severely reduce your total number of users, require you to increase your subscription price, or require that your marketing budget is increased to gain the same number of users to be profitable.

Excessive government internet restrictions

In China, the internet is heavily restricted and most websites on the internet is blacklisted by default. The government decides which websites are allowed to be viewed and these are added to the whitelist that users can access. If you were working on a project and were planning on deploying it in China, then your project could be heavily effected or even totally banned by new government regulations. Additionally, already established websites have a significant advantage compared to new competitors. Alibaba and Baidu enjoy great benefits from restricted internet access as international companies are totally barred from entering the market and competing. The restrictions can also have the opposite effect and pose an advantage for new competitors. In 2020 there is increasing worry that the Chinese company Zoom is capable of spying on users meetings. This means that companies, especially in the USA, may refuse to use the Zoom service if they are discussing very sensitive data or corporate secrets. This provides a great opportunity for a US company to provide a similar, more secure service to replace Zoom in the USA.

1.7 Describe which are the appropriate filetypes for websites

Appropriate filetypes for websites include:

HTML files ‚Äì Hyper Text Markup Language. This is the skeleton of a website.

CSS files ‚Äì Cascading Style Sheets. These give you the ability to change the visual elements in the html code. I have used this in my website to change the background colour, font types, sizes and colour and also to set a theme for my navigation bar.

PNG files ‚Äì PNG files are image files that allow transparency. You can see these used throughout my website. The transparency allows the image to be displayed over the top of the dark background without having any white ‚Äúsquare‚Äù background behind it.

JPG files ‚Äì JPEG files are best for photos, as they allow the image to be compressed without ruining the image. My website only used simple graphics, to keep the file size small, and therefore I have chosen not to use JPEG on this project.

GIF ‚Äì GIF files allow for images that have moving animation.

MP3 ‚Äì MP3 files are audio files that have been compressed. They are very useful for web design. I have no included any MP3 files, but I would like to add some at a later date if I add some animated GIFS.
